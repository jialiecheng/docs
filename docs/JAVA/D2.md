# javaSE学习(进制、数据类型、变量)
## 02.07_Java语言基础（原码补码反码）
-  原码
二进制定点表示法，最高位为符号位，0表示正，1表示负，其余表示数值大小

如下通过一个字节码也就是8个二进制位表示+7和-7（计算机在进行运算的时候都是以补码的形式运算）

0（符号位）0000111

1（符号位）0000111

- 反码
正数的反码与其原码相同，负数的反码是其原码的逐位取反，但符号位除外

- 补码
正数的补码与其原码相同，负数的补码在其反码的末位加1

## 02.10_Java语言基础（数据类型的概述和分类）
- Java语言是强类型语言，对于每一种数据都定义了明确的具体数据类型，在内存中分配了不同大小的内存空间
- 有基本数据类型和引用数据类型
- 基本数据类型有四种：
整数型：byte 1字节 -128—127，short 2字节 -2^15—2^15-1，int 4字节 -2^31—2^31-1，long 8字节 -2^63—2^63-1
浮点型 ：float 4字节， double 8字节
字符型：char 2字节 0—65535（没有负数）
布尔型：理论上1/8字节但Java中没有明确指定
 

## 02.13_Java语言基础（数据类型隐式转换和强制转换）
- 隐式转换：例如当int类型与byte类型相加时，byte类型会先自动转换为int类型再运算
- 强制转换：例如若int类型想赋给一个byte类型，则在值前面加（byte）

- 强制类型转换时，如果超出了被赋值数据类型的范围，得到的结果会与期望的结果不同，因为会砍掉前面的部分
byte与byte、short、char两个变量进行运算的时候会自动提升为int类型，因为默认的就是int类型，两个变量存储的值是变化的，在编译时无法判断里面具体的值，运算有可能会超出即损失精度；如果是常量运算就不会，因为Java编译器有常量优化机制

## 02.16_Java语言基础（基本数据类型）
- 进行混合运算的时候byte、short、char不会相互转化，都会自动提升为int类型，其他类型进行混合运算是小的数据类型提升为大的：byte、short、char——int——long——float——double
- float表示的数据范围比long的数据范围大

## 02.17_Java语言基础（字符和字符串）
- 任何数据类型用+与字符串相连接都会产生新的字符串
- Java语言中的char可以存储一个汉字，因为Java语言采用的是Unicode编码，Unicode编码中每个字符占两个字节，一个中文也是占两个字节，所以char可以存储一个中文
- 若把一个数字赋予char类型则打印的是ASCII表中对应的字符

## 02.19_Java语言基础（运算符）
- 运算符分为：算数运算符、赋值运算符、比较（关系或条件）运算符、逻辑运算符、位运算符、三目（元）运算符
- +号在Java中有三种作用：正好、加法运算、字符串的连接符
- 整数相除只能得到整数，如果想得到小数必须变化为浮点型
- 任何一个正整数%2不是0就是1可以用来当做切换条件
- ++、--运算符：单独使用时放在操作数前面和后面效果一样；参与运算使用时放在前面先自增或自减再运算，放在后面先运算再自增或自减
- short s=1； s=s+1会报错因为short与int运算时会自动转换成int类型；s+=1不会报错因为会强制转换为short
